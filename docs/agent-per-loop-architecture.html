Agent per Loop — Architecture

1 loop = 1 LLM call (with tool interleaving) — Surpassing IDE Agent × Hermes

Outer Loop (1 turn = 1 LLM call)

  • • User message → Context Assembly → LLM Call
  • • LLM returns text or tool_calls or both
  • • No tool_calls → deliver final response immediately
  • • Has tool_calls → enter inner loop, then re-LLM
  • • Deliver → Memory persist → Next turn

Inner Loop (tool call sub-loop)

  • • LLM requests tool execution: tool_calls[]
  • • Each tool runs: read_file, terminal, web_search, etc.
  • • Tool results appended to conversation as new messages
  • • Back to LLM with updated context (still same turn)
  • • Repeats until LLM returns final text response

Sidecar Infrastructure

  • • Knowledge Graph — semantic code index (SQLite)
  • • File Watcher — real-time AST parsing
  • • Sandbox — Docker/Direct code execution
  • • Security Monitor — SAST scanning on every change
  • • LLM Router — privacy detector → model select → fallback
